1. 知识库负责详细讲解概念、机制、链路、边界和工程权衡。
2. 题库必须基于知识库生成，不能孤立堆题。
3. 实践资料可以进入主线知识库，但前端不单独展示来源品牌。
4. 协议、API、版本行为和产品能力必须优先使用官方来源复核。
5. 每次新增或迁移内容后必须重建 catalog，并执行校验、审计和前端构建。
