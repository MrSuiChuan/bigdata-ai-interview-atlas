# 页面风格方向

这个系统不是品牌官网，也不是单页产品展示，而是一个长期演进的知识系统。因此视觉方向要优先服务这三件事：

1. 中文长文阅读
2. 结构化知识导航
3. 后续题库与练习界面的统一扩展

## 风格定位

推荐采用下面这套组合方向：

1. 以 `Mintlify` 的信息清晰度作为基础
2. 吸收 `Notion` 的温和留白与阅读感
3. 局部借鉴 `ClickHouse` 的技术强调与模块秩序

不建议把深色终端风格作为主文档风格，因为它更适合 Agent 演示页或工具操作页，不适合大量中文长文阅读。

## 视觉原则

### 1. 文档优先，不走营销页套路

首页应该是一张知识地图，而不是传统 SaaS 营销页。重点是让用户快速理解：

1. 这里有什么内容
2. 内容怎么组织
3. 应该从哪里开始

### 2. 浅色主基调

默认使用浅色、纸感背景和温和阴影，让中文正文更易读，也更适合长时间学习。

### 3. 技术感点到为止

技术感主要放在：

1. 结构卡片
2. 数据块
3. 路径图
4. 来源与版本标签

不要把整站做成过强的未来科技风，否则会伤害可读性。

## 配色方向

### 主色

使用偏绿色的主色，表达可信、系统性和知识沉淀感。

### 辅色

用少量琥珀黄做提示色，适合：

1. 重点提醒
2. 层级分隔
3. 标签和引导

### 背景

背景不使用纯平白色，建议保留轻微纸感、渐变和低对比纹理。

## 排版方向

### 标题

标题使用更有人文感的衬线风格，让知识系统不显得冰冷。

### 正文

正文优先保证中文可读性，采用清晰的无衬线字族，并保留英文术语的技术辨识度。

### 代码与术语

代码、字段名、命令、协议关键字全部使用等宽字体，和正文形成明确区分。

## 组件语言

系统中应长期复用的组件包括：

1. 来源卡片
2. 面试重点卡片
3. 常见误区卡片
4. 追问链卡片
5. 学习路径卡片
6. 结构关系卡片

这些卡片要统一圆角、边框强度、阴影级别和标题节奏，避免页面像拼装站。

## 页面层级建议

### 首页

首页做成知识地图，重点展示：

1. 大数据主线
2. AI Agent 主线
3. 系统结构
4. 学习路径

### 文档页

文档页以阅读为主，重点是：

1. 正文舒适
2. 侧边导航清晰
3. 目录区稳定
4. 代码块、表格、引用块容易扫描

### 题库页

题库页比文档页更紧凑，强调：

1. 难度
2. 题型
3. 掌握度
4. 追问链
5. 标准答案和评分点

### AI 页

AI 模拟面试页可以比文档页更有未来感，但仍然应保持整体配色和结构一致，不应该完全像另一个产品。

## 设计约束

1. 中文必须是主语言
2. 英文只用于专业术语和代码语义
3. 不默认走深色模式主视觉
4. 不堆砌炫技动画
5. 优先保证桌面和移动端都可读

## 当前落地方向

当前文档站已经开始按这套方向演进：

1. 浅色纸感背景
2. 绿色主色
3. 知识地图式首页
4. 结构化卡片和学习路径入口
5. 面向中文长文的文档页阅读样式

后续题库页、搜索页、练习页都应继续沿用这套视觉系统。
