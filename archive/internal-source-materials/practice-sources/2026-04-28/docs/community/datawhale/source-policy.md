# 来源等级

Datawhale 在本系统中使用 `source_tier: trusted-community`。它高于普通社区资料，但不替代官方文档和论文。

# 可以直接采用的内容

1. 学习路线。
2. 项目实践经验。
3. 工程案例拆解。
4. 常见坑点和实操建议。
5. 面试题素材和项目复盘素材。

# 需要交叉复核的内容

1. API 行为。
2. 协议语义。
3. 框架版本差异。
4. 模型能力边界。
5. 性能指标和资源消耗结论。
6. 安全边界和生产可用性判断。

# 整理规则

1. 每篇 Datawhale 整理页必须写清来源项目。
2. 每篇整理页必须说明它对应的主知识库知识点。
3. 每个项目至少拆出核心概念、实践路径、可面试问题和常见误区。
4. 如果多个项目覆盖同一主题，以融合后的主题页为主，不重复堆内容。
5. 如果结论只来自社区经验，必须标注 trusted-community，不得伪装成官方事实。

# 状态定义

1. `source-indexed`：只完成来源登记。
2. `mapped`：完成仓库到知识点映射。
3. `digested`：完成专区整理页。
4. `merged-to-main-kb`：已经融合进主知识库。
5. `question-ready`：已经生成题目并进入前端题库。
6. `official-cross-checked`：关键事实完成官方复核。
