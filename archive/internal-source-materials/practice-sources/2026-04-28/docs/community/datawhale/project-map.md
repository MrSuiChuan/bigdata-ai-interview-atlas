# 一句话定位

项目地图用于把 Datawhale 仓库从“项目列表”转成“面试知识体系”。它只记录整理后的主题关系，不照搬原仓库目录。

# P0 AI Agent 主线

1. `hello-agents`：Agent 基础、工具调用、任务循环、项目实践。
2. `agentic-ai`：Agentic AI 思路、规划、工具使用、多步执行。
3. `agent-tutorial`：Agent 入门路径和基础题材。
4. `self-harness`：长任务 Agent 的 harness engineering、恢复、观测、评估。
5. `handy-multi-agent`：多 Agent 协作、角色划分、通信和协同边界。
6. `hugging-multi-agent`：MetaGPT 类多 Agent 工程实践。
7. `easy-langent`：LangChain、LangGraph、Lagent 等框架实践。
8. `agent-skills-with-anthropic`：Agent skills、工具使用、上下文工程。

# P0 RAG 主线

1. `all-in-rag`：RAG 全链路，覆盖文档处理、Embedding、检索、重排、生成和评估。
2. `wow-rag`：RAG 框架和项目实践。
3. `llm-universe`：大模型应用开发、RAG 应用、知识库问答。
4. `what-is-vs`：向量检索和 RAG 实践。
5. `easy-vecdb`：向量数据库基础和使用边界。

# P0 大模型主线

1. `happy-llm`：大模型基础学习路线。
2. `base-llm`：从 NLP 到 LLM 的算法全栈。
3. `self-llm`：开源大模型部署、微调和本地实践。
4. `diy-llm`：预训练、Tokenizer、Transformer、MoE、系统训练实践。
5. `code-your-own-llm`：白盒实现和端到端训练理解。
6. `tiny-universe`：小模型和训练系统实践。
7. `post-training-of-llms`：SFT、偏好优化、后训练流程。
8. `llm-deploy`：推理和部署理论与实践。
9. `llms-from-scratch-cn`：从零构建大语言模型。

# P1 工程实践主线

1. `self-dify`：Dify 本地部署和应用搭建。
2. `handy-ollama`：Ollama 本地模型部署。
3. `coze-ai-assistant`：Coze 工作流和 AI 助手实践。
4. `mcp-lite-dev`：MCP 极简开发实践。
5. `llm-protocols-guide`：LLM 协议学习。
6. `smart-dev`：AI Coding 工作流。
7. `vibe-blog`：多 Agent 写作和内容生成案例。
8. `easy-vibe / vibe-vibe`：Vibe Coding 与 AI 应用开发路径。

# 整理输出要求

每条主线最终都要输出四类资产：

1. Datawhale 专区整理页。
2. 主知识库融合页。
3. 面试题。
4. Claim 和来源映射。
