# 题目

n8n 的 Credentials 和 Error Workflow 为什么是上线前必须讲清的两个对象？

# 一句话结论

因为一个决定外部系统能以什么身份被访问，另一个决定失败后系统会往哪里去，这两层共同决定流程是否安全、是否可控。

# 核心机制

1. Credentials 决定外部系统连接身份。
2. Error Workflow 决定失败后的兜底路径。
3. 两者都属于生产治理，而不是可有可无的辅助功能。
4. 自定义节点的 contract 要和它们协同设计。

# 标准答案

Credentials 解决的是 workflow 访问外部系统时的身份注入和最小权限问题；Error Workflow 解决的是节点失败后系统究竟要重试、告警、补偿还是停止。如果这两层没设计好，一个流程即使主路径跑通，也可能因为凭证误配而越权，或者因为错误路由不清而无限重试。对引入 AI 节点的流程来说，这一点更重要，因为模型和外部 API 都会增加失败面和权限面。

# 必答点

1. 说明 Credentials 与业务授权的区别。
2. 说明 Error Workflow 与事务保证的区别。
3. 说明为什么 AI workflow 会放大这两层的重要性。
4. 说明 Custom Node 需要遵守同样 contract。
5. 说明失败不是都应该自动重试。

# 常见误答

1. 认为凭证配好就等于权限安全。
2. 所有错误都交给重试。
3. 不讲自定义节点的 contract。
4. 不区分连接身份和业务授权。
