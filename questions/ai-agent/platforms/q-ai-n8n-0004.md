# 题目

n8n 流程中“AI 节点输出不稳定”时，应该优先排哪几层？

# 一句话结论

先排 Trigger 和上游数据，再排 Expressions 和参数拼装，再排 AI 节点本身，最后才排 Error Workflow 与外部系统写入副作用。

# 核心机制

1. 上游数据错，后面再强的模型也会歪。
2. Expressions 经常是参数错位的根源。
3. AI 节点只是局部决策点，不是唯一问题源头。
4. Error Workflow 决定异常是否被进一步放大。

# 标准答案

n8n 里 AI 节点输出不稳定，排障顺序最好从上游往下游看。先确认 Trigger 和前置节点是否产生了正确输入；再看 Expressions 是否把字段正确映射到了 AI 节点参数；然后再看 prompt、memory 或 AI 节点内部配置本身；最后检查 Error Workflow 和后续写入节点是否把不稳定结果放大成更大问题。这样排的原因是，AI 节点经常只是暴露问题的地方，真正根因可能在输入准备层。

# 必答点

1. 说明先排上游输入。
2. 说明 Expressions 的高风险性。
3. 说明 AI 节点不是唯一根因。
4. 说明 Error Workflow 会放大或抑制问题。
5. 说明排障顺序要沿数据路径走。

# 常见误答

1. 一上来只调 prompt。
2. 不检查输入字段。
3. 忽略参数映射错误。
4. 不看错误路径设计。
