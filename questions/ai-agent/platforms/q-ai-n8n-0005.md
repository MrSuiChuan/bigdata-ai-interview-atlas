# 题目

n8n AI workflow 上线后，哪些指标比“流程成功率”更早暴露工程问题？

# 一句话结论

节点级耗时、重试路径频率、外部 API 错误率、AI 节点 token 成本和 trigger 队列深度，往往比最终成功率更早显示系统正在恶化。

# 核心机制

1. 节点级指标能暴露局部瓶颈。
2. 重试路径频率能暴露失败放大。
3. 外部 API 错误率能暴露集成层问题。
4. token 成本能暴露 AI 节点不稳定或输入膨胀。
5. trigger 队列深度能暴露吞吐瓶颈。

# 标准答案

流程成功率通常是比较滞后的指标。更早的预警信号通常来自节点级 P95 耗时、Error Workflow 触发频率、外部 API 错误率、AI 节点 token 成本、平均重试次数和 trigger 队列深度。如果这些指标持续恶化，即使总成功率暂时还没掉下来，也往往说明工作流已经在输入、集成或成本层面开始失控。尽早观察这些中间信号，能更早地发现工程问题。

# 必答点

1. 说明节点级耗时。
2. 说明重试路径频率。
3. 说明外部 API 错误率。
4. 说明 AI token 成本。
5. 说明 trigger 队列深度。

# 常见误答

1. 只看最终流程成功率。
2. 不看 Error Workflow 触发频率。
3. 不看节点级成本。
4. 不看外部集成层。
