# 题目

A2A 为什么要把 task、`input-required` 和 subscription 设计成正式协议语义，而不是让客户端自己约定？

# 一句话结论

因为远程 Agent 协作不是一次调用拿结果，协议必须把长任务推进、续跑和一致观察正式建模。

# 标准答案

A2A 面向的是远程 Agent 协作，而不是本地函数调用。很多协作天然是长任务，可能需要等待更多输入、等待人工确认，或者在远端持续推进。如果这些行为只靠客户端和服务端私下约定，系统就很难跨实现互操作。A2A 把 task 作为核心对象，把 `input-required` 作为正式中断态，把 subscription 作为一致观察机制，就是为了把“继续同一任务”“知道任务当前真实状态”“区分失败和续跑”这些关键语义放进协议本身。只有这样，远程 Agent 才能稳定协作，而不是退化成不太可靠的 RPC。

# 必答点

1. A2A 处理的是长任务协作
2. task 是正式协作单元
3. `input-required` 代表续跑，不等于失败
4. subscription 用于一致观察任务状态

# 常见误答

1. 这些都是客户端自己定义就行
2. `input-required` 就是报错
3. 只要能拿到最终结果就够了
4. 不区分长任务和单次调用
