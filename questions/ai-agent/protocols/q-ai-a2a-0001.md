# 题目

为什么 A2A 应该被讲成 Agent-to-Agent 协议，而不是工具协议？

# 一句话结论

因为它面向的是一个 Agent 系统与另一个独立 Agent 系统协作，而不是本地 Agent 调一个工具函数。

# 核心机制

1. 远端对象是 opaque agent system
2. 协作围绕 task 展开
3. 协议层明确采用 JSON-RPC over HTTP(S) 与流式回传

# 标准答案

A2A 之所以是 Agent-to-Agent 协议，是因为它面向的是两个相对独立的 Agent 系统之间的协作。远端 agent 可以是 opaque 的，不需要暴露内部 memory、tools 或 planning 细节；双方围绕 task 进行上下文交换和长运行协作。这和工具协议不一样，工具协议更像本地 Agent 接一个可执行能力，而 A2A 连接的是另一个 Agent 系统。

# 必答点

1. opaque remote agent
2. task-based collaboration
3. not tool protocol

# 常见误答

1. 把 A2A 说成远程 tool calling
2. 把 A2A 当成编排框架
