# 题目

A2A 和 MCP 的关系应该怎么讲才准确？

# 一句话结论

MCP 更适合接工具和上下文，A2A 更适合连远端 Agent 系统；它们是互补关系，不是替代关系。

# 核心机制

1. MCP 面向 tools/resources/prompts
2. A2A 面向 remote agents 和 tasks
3. Agent Card 是 A2A 发现远端 Agent 的关键元数据对象

# 标准答案

A2A 和 MCP 不在同一层。MCP 更适合把 tools、resources、prompts 接给 Agent，A2A 更适合把一个 Agent 系统接到另一个 Agent 系统上。官方文档也明确说两者是 complementary 的。A2A 还通过 Agent Card 做 discovery，并建议使用 `/.well-known/agent.json` 暴露远端 agent 的身份、skills、capabilities 和鉴权信息。所以高质量回答应该是“本地能力接入可用 MCP，跨 Agent 系统协作用 A2A”。

# 必答点

1. complementary not replacement
2. MCP for tools
3. A2A for agents and discovery

# 常见误答

1. 说 A2A 会取代 MCP
2. 把 Agent Card 讲成普通介绍页
